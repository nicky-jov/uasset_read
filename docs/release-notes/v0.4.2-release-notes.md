# v0.4.2 发布说明

**发布日期**: 2026-06-04
**Python 要求**: 3.10+
**分支**: `0.4.2-dev`

## 重大变更

### 架构重构
- **IR → Renderer 架构** — 取代旧的 exporter 注册表，建立统一中间表示（PackageIR）+ 多渲染器架构
- **CLI 与核心逻辑分离** — `core.py` 提供纯函数 API（`parse_single()`, `parse_batch()`, `list_formats()`），CLI 仅负责参数绑定
- **废弃模块清理** — 移除 `exporter/`、`n2c/`、`agent/` 等废弃模块

### 新增功能
- **偏移/范围诊断** — `OffsetRangeDiagnostic` 模型，所有越界异常现在结构化报告
- **C++ 输出质量提升**:
  - 变量名 sanitizer（空格 → 下划线，非法字符移除）
  - 空默认值修复（不再输出 `= ;`）
  - 函数体 wrapper 嵌套修复
  - 方法实现加 `ClassName::` 作用域前缀
  - include 去重
- **Kismet 反编译语义提升**:
  - deprecated/instrumentation token 静默跳过
  - `FunctionRefResolver` 函数引用增强（≥80% 解析率）
  - `BlueprintNodeCleaner` 常见蓝图节点语义映射
  - 控制流结构化增强（for/switch/sequence 模式检测）
  - 补齐 extract_cpp_functions 遗漏的反编译函数（第三条路径）
  - body_text 注入时执行符号映射替换
- **8 种输出格式** — json, json_summary, cpp_skeleton, markdown, text, text_summary, blueprint_text, blueprint_ue_text
- **质量统计脚本** — `scripts/quality_stats.py` 扫描 C++ 输出质量指标，支持 `--json`、`--verbose`、`--threshold` 选项
- **截断/损坏文件诊断** — 使用 `OffsetRangeDiagnostic` 结构化报告截断/损坏文件
- **变量分类过滤** — 过滤元数据变量，不混入 PackageIR.variables
- **构造函数元数据过滤** — 不注入 BlueprintSystemVersion 等内部字段
- **PropertyFallback 系统** — 未知属性返回 PropertyFallback 带诊断信息而不是失败
- **ClassHandlerRegistry** — 每类序列化带可配置回退策略

### Bug 修复
- Pin GUID 格式统一为 32 位小写 hex（无 dashes）
- PinReference GUID 格式统一 - 移除冗余代码和死字段
- 裸数字修复
- goto label 修复
- 空函数体充实

### 测试
- **994 个测试通过**（2 xfail），覆盖 12+ 种资产类型
- 新增 quality_stats.py 测试
- 新增 UE4 legacy 版本兼容性测试
- 关键测试添加 @pytest.mark.regression 标记
- 稳定资产在 strict 和 tolerant 双模式下通过

### 内部改进
- 集中定义 BLUEPRINT_METADATA_KEYS 到 constants.py
- 移除 quality_stats.py 中的冗余零文件警告

## 兼容性
- **仅支持未烘焙/编辑器保存的资产**
- **UE5 资产** — legacy_file_version ∈ {-9, -8}
- **UE4 资产** — 不支持（产生明确的版本错误）

## 已知限制
- ParticleSystem (P_Fire.uasset) 使用 UE4 legacy_file_version=-3，标记为 xfail
- Cooked 资产的图数据已被剥离，无法解析
